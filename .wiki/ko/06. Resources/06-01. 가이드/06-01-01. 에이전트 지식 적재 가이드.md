# 에이전트 지식 적재 가이드

이 문서는 OMW base wiki에 에이전트 산출물과 작업 지식을 남길지 판단하는 기준을 정의합니다. 특정 제품, 회사, 고객, 개인에게 종속된 내용은 공개 base wiki에 남기지 않습니다.

## 적재 기준

- 반복해서 재사용할 수 있는 규칙, 절차, 판단 기준만 장기 노트 후보로 봅니다.
- 개인 메모, 팀 내부 결정, 고객명, 계정, 토큰, 비공개 URL, 미공개 제품 세부사항은 base wiki에 적재하지 않습니다.
- 원문 로그와 회의 전문은 Raw에 임시 보관할 수 있지만, 공개 배포 전에는 민감정보와 조직 고유 맥락을 제거합니다.
- 확정되지 않은 의견은 `01. Inbox`에 두고, 검증된 기준만 `03. Permanent Notes` 또는 `06. Resources`로 승격합니다.

## 작업 순서

1. 입력 자료가 공개 가능한 범용 지식인지 확인합니다.
2. 자료의 유형을 Raw, Literature, Permanent, Project, Area, Resource 중 하나로 정합니다.
3. 필수 frontmatter와 출처, 민감정보 검사 상태를 기록합니다.
4. 장기 노트로 승격할 때는 사실, 기준, 예외, 검증 방법만 남깁니다.
5. 기존 지도와 카탈로그에 연결하여 다음 사용자가 탐색할 수 있게 합니다.
